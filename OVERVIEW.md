# MemoHub - 整体功能和思路

## 📌 项目定位

**MemoHub** 是一个**智能双轨记忆系统**，作为"我的记忆中心"，为用户提供：

- 🧠 **通用知识管理**（GBrain）
- 💻 **代码记忆管理**（ClawMem）
- 🔍 **语义搜索**（基于向量嵌入）
- 🔒 **本地隐私保护**
- 🚀 **快速响应**

---

## 📊 功能模块

### 1. GBrain（通用知识轨道）

#### 功能
- ✅ 添加知识
- ✅ 语义搜索知识
- ✅ 删除知识
- ✅ 查看统计信息

#### 数据结构
```typescript
interface GBrainRecord {
  id: string;           // 唯一标识
  text: string;         // 知识内容
  category: string;      // 分类
  tags: string[];        // 标签
  importance: number;    // 重要性评分
  embedding: number[];   // 向量表示
  createdAt: string;    // 创建时间
}
```

#### 分类体系
- `user` - 用户偏好
- `memory` - 系统记忆
- `system` - 系统配置
- `project-catalog` - 项目目录
- `environment` - 环境信息
- `other` - 其他

#### 使用场景
- 存储用户偏好（如"喜欢 TypeScript"）
- 记录环境信息（如"使用 Node.js v22"）
- 保存项目信息（如"项目是双轨记忆系统"）
- 管理系统配置（如"Ollama 服务地址"）

---

### 2. ClawMem（代码记忆轨道）

#### 功能
- ✅ 添加代码片段
- ✅ 语义搜索代码
- ✅ 查看统计信息

#### 数据结构
```typescript
interface ClawMemRecord {
  id: string;           // 唯一标识
  text: string;         // 代码内容
  filePath: string;     // 文件路径
  symbolName: string;   // 符号名
  astType: string;      // AST 类型
  language: string;     // 编程语言
  tags: string[];       // 标签
  importance: number;    // 重要性评分
  embedding: number[];   // 向量表示
  createdAt: string;    // 创建时间
}
```

#### AST 类型识别
- `interface` - 接口定义
- `function` - 函数
- `class` - 类
- `type` - 类型别名
- `variable` - 变量
- `unknown` - 未知

#### 编程语言支持
- TypeScript
- JavaScript
- Python
- 其他（可扩展）

#### 使用场景
- 存储常用代码片段
- 记录 API 使用方式
- 保存架构设计模式
- 管理工具函数库

---

### 3. 向量嵌入（Embedder）

#### 功能
- ✅ 生成文本向量表示
- ✅ 批量嵌入
- ✅ 支持多种嵌入模型

#### 技术栈
- Ollama（本地 AI 运行时）
- nomic-embed-text-v2-moe（嵌入模型）
- 768 维向量

#### 工作流程
```
文本输入 → Ollama API → 生成向量 → 存储到 LanceDB
```

---

### 4. LanceDB（向量数据库）

#### 功能
- ✅ 高效向量存储
- ✅ 相似度搜索
- ✅ 大规模数据支持

#### 特点
- 本地存储（无需云端）
- 高性能（毫秒级查询）
- 灵活的数据结构
- 支持过滤和排序

### 6. 配置管理

#### 配置层次
1. **YAML 配置文件**（config/config.yaml）
2. **环境变量**（可覆盖 YAML）

#### 配置项
```yaml
# 嵌入模型配置
embedding:
  model: "nomic-embed-text-v2-moe"
  baseURL: "http://localhost:11434"
  dimension: 768

# 数据库配置
gbrain:
  dbPath: "~/.memohub/gbrain.lancedb"
  tableName: "gbrain"

clawmem:
  dbPath: "~/.memohub/clawmem.lancedb"
  tableName: "clawmem"

# 同步配置（私有仓库）
sync:
  enabled: false
  repoUrl: ""
  branch: "main"
  syncInterval: "1h"
```

## 🎯 技术思路

### 1. 双轨分离设计

**为什么分离？**

- **GBrain**：存储事实性知识，用户偏好，系统配置
- **ClawMem**：存储技术性代码，API 使用，架构设计

**优势**：
- 针对性优化
- 独立管理
- 灵活扩展

### 2. 向量嵌入搜索

**为什么用向量？**

- 语义理解（不只是关键词匹配）
- 相似度排序
- 支持模糊查询

**技术选择**：
- Ollama（本地运行，隐私保护）
- nomic-embed-text-v2-moe（高质量嵌入）
- LanceDB（高效向量数据库）

### 3. 本地优先

**为什么本地？**

- 隐私保护（数据不上传）
- 快速响应（无网络延迟）
- 离线可用
- 成本控制（无需 API 费用）

### 4. 兼容性设计

**与 Hermes 兼容**：
- 共享同一数据库
- 配置文件灵活
- 数据格式一致

**优势**：
- 无缝集成
- 数据迁移容易
- 多工具共存

### 5. 扩展性设计

**插件系统**：
- 自动记忆提取
- 记忆注入
- 可扩展架构

**未来可能**：
- Web 界面
- 移动应用
- 多用户支持
- API 服务

MemoHub 是一个**完整的双轨记忆系统**，具备：

5. **灵活配置**：YAML + 环境变量
6. **扩展性**：插件系统，可扩展架构
7. **兼容性**：与 Hermes 等工具集成
8. **安全性**：本地存储，私有仓库同步

**核心价值**：作为"我的记忆中心"，帮助用户高效管理和检索知识和代码。
