## 📌 项目定位

**MemoHub** 是一个 **Agent 记忆内核**，基于 **多轨道动态矩阵架构**，旨在成为像真实"数字大脑"一样的记忆系统：

- 🧠 **逻辑沉淀轨道** (track-insight) - 存储 LLM 提纯后的事实和决策
- 💻 **静态资产轨道** (track-source) - 处理源代码和 AST 解析
- 🌊 **时序流轨道** (track-stream) - 记录原始对话 Log（计划中）
- 📚 **真理库轨道** (track-wiki) - 存储经过确认的结构化知识（计划中）
- 🔧 **治理轨道** (track-librarian) - 去重、蒸馏、澄清
- 🔍 **语义搜索** - 基于向量嵌入的精准召回
- 🔒 **本地隐私保护** - 数据不上传第三方
- 🚀 **快速响应** - 毫秒级查询响应

## 📊 核心轨道详解

### 1. Track: Source（静态资产轨道）✅

**职责**: 处理源代码，利用 Tree-sitter 进行 AST 解析

**核心功能**:
- ✅ 代码文件解析和存储
- ✅ AST 语法树提取
- ✅ 符号（函数、类、变量）索引
- ✅ 调用链实体记录
- ✅ 工程级精准召回

**CLI 命令**:
```bash
memohub add-code ./src/utils.ts -l typescript
memohub search-code "fetch function"
```

**MCP 工具**:
```typescript
add_code({ code, language, file_path })
search_code({ query, limit })
```

---

### 2. Track: Insight（逻辑沉淀轨道）✅

**职责**: 存储 LLM 提纯后的事实、决策定论和用户偏好

**核心功能**:
- ✅ 事实性知识存储
- ✅ 用户偏好记录
- ✅ 决策定论保存
- ✅ 系统价值观维护

**CLI 命令**:
```bash
memohub add "用户喜欢 TypeScript" -c user -i 0.9
memohub search "开发偏好"
```

**MCP 工具**:
```typescript
add_knowledge({ text, category, importance, tags })
query_knowledge({ query, limit })
```

---

### 3. Track: Stream（时序流轨道）🚧

**职责**: 记录原始对话 Log

**核心功能**:
- ✅ 对话历史存储
- ✅ 时序数据记录
- ✅ TTL（生存时间）管理
- ✅ 自动蒸馏和清理

**TTL 机制**:
- **默认 TTL**: 30 天
- **蒸馏后删除**: 由 Librarian 蒸馏到 Insight 轨后删除
- **保持轻量**: 避免系统臃肿

---

### 4. Track: Wiki（真理库轨道）🚧

**职责**: 存储经过 Librarian 整理且通过"澄清环节"确认的结构化知识

**核心功能**:
- ✅ 结构化知识存储
- ✅ 澄清环节确认
- ✅ 真理库维护
- ✅ 知识版本管理

**澄清环节**:
1. 知识提取
2. 冲突检测
3. 人工确认
4. 版本发布

---

### 5. Track: Librarian（治理轨道）✅

**职责**: 知识治理、去重、蒸馏、澄清

**核心功能**:
- ✅ 去重扫描（dedup）
- ✅ 知识蒸馏（distill）
- ✅ 澄清环节（clarify）
- ✅ 版本管理（version）

**CLI 命令**:
```bash
memohub dedup --track track-insight
memohub distill --track track-insight
```

## 🎨 架构设计三大原则

### 1. 灵肉分离与 CAS 去重

**原则**: 无论增加多少个轨道，底座永远是 `storage-flesh`

**优势**:
- ✅ **物理去重**: 同一段内容只存储一份
- ✅ **跨轨道共享**: Source 轨和 Wiki 轨可共享同一段代码
- ✅ **存储优化**: 节省磁盘空间
- ✅ **索引一致**: 通过 hash 关联，不会失效

---

### 2. 模型解耦（AI-Provider）

**原则**: 所有的轨道逻辑严禁直接调用 OpenAI 或 Ollama 的 SDK

**优势**:
- ✅ **厂商无关**: 随时更换 AI 模型
- ✅ **成本优化**: 动态选择最优模型
- ✅ **代码稳定**: 模型切换不影响轨道逻辑

---

### 3. Librarian 的权力边界

**原则**: Librarian 可以执行"合并"和"归档"，但遇到逻辑冲突必须触发"澄清环节"

**权力范围**:
- ✅ **合并（Merge）**: 合并重复或相似的知识
- ✅ **归档（Archive）**: 将过期数据归档
- ✅ **蒸馏（Distill）**: 从 Stream 提炼到 Insight
- ❌ **自行裁决**: 遇到业务逻辑冲突必须暂停

## 🔧 技术栈

### 核心技术

- **语言**: TypeScript 5.9.3（遵守 Airbnb 规范）
- **运行时**: Node.js >= 22.0.0, Bun >= 1.3.0
- **包管理**: Bun Workspace Monorepo
- **数据库**: LanceDB 0.26.2（向量存储）
- **嵌入**: Ollama (nomic-embed-text-v2-moe)
- **协议**: MCP (Model Context Protocol)

### 代码规范

**注释规范**:
- ✅ 所有代码必须添加汉语注释
- ✅ 详细解释每个功能和文件的作用
- ✅ 类和函数必须有 JSDoc 注释

```typescript

  id = 'track-insight';

  /**
   * 添加知识到轨道
   * @param instruction - Text2Mem 指令
   * @returns 添加结果
   */
  async execute(instruction: Text2MemInstruction): Promise<Text2MemResult> {
    // 实现逻辑...
  }
}
```

**Airbnb 规范**:
- ✅ 使用 2 空格缩进
- ✅ 使用单引号
- ✅ 使用分号
- ✅ 使用 const/let，不使用 var
- ✅ 函数命名使用 camelCase
- ✅ 类命名使用 PascalCase

**输出规范**:
- ✅ **0.1** 所有输出必须使用中文
- ✅ **0.2** 专有名词允许保留原文（API, MCP, AST）

## 📚 总结

MemoHub v1 是一个 **Agent 记忆内核**，采用 **多轨道动态矩阵架构**：

1. ✅ **多轨道**: Source、Insight、Stream、Wiki 等
2. ✅ **语义搜索**: 基于向量嵌入的智能检索
3. ✅ **本地优先**: 隐私保护，快速响应
4. ✅ **CLI + MCP**: 双重接口，灵活集成
5. ✅ **灵活配置**: YAML + 环境变量
6. ✅ **可扩展性**: 动态轨道注册，插件化架构
7. ✅ **标准协议**: Text2Mem 协议，12 个原子操作
8. ✅ **治理能力**: 去重、蒸馏、澄清

**核心价值**: 从"存储器"进化到"认知操作系统"
